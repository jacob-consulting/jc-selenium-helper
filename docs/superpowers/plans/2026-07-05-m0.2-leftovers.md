# M0.2 Leftovers Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the two remaining M0.2 test gaps (`hover_with_offset`, the `jc_browser` fixture) and, bundled with the first, apply the M3.2 source fix that simplifies `hover_with_offset` to a single chained action.

**Architecture:** Three small, independent changes. Task 1 simplifies `hover_with_offset` and adds a real headless-Chrome behavior test driven by a new deterministic HTML fixture. Task 2 adds a fast `pytester` test for the trivial `jc_browser` wiring using a stubbed `selenium` fixture (no browser launch). Task 3 records the change in the CHANGELOG and verifies the full suite.

**Tech Stack:** Python 3.12+, Selenium 4.38+ (`ActionChains`), pytest + pytest-selenium, pytester, ruff.

## Global Constraints

- `requires-python >= 3.12`; supported matrix is 3.12 / 3.13 / 3.14 (keep code compatible with all three).
- Runtime dependency is `selenium>=4.38` only; test-only helpers live behind the `test` extra. Do not add new dependencies.
- Locators default to XPath; pass `by=By.CSS_SELECTOR` for CSS. `Browser.__init__(self, driver, default_timeout=120, poll_pause=1)` stores the driver as `self.driver`.
- No release / version bump in this plan. CHANGELOG gains an **unreleased** entry only.
- Follow existing test style: assert real observable behavior, not just "does not raise". HTML fixtures live in `tests/fixtures/`.
- Reference spec: `docs/superpowers/specs/2026-07-05-m0.2-leftovers-design.md`.

- [ ] **Step 4: Apply the M3.2 fix â€” single chained action**

In `src/jc_selenium_helper/browser.py`, replace the two-`perform()` body:

```python
    def hover_with_offset(self, locator: str, x_offset: int, y_offset: int, by: str = By.XPATH) -> None:
        element = self.find(locator, by)
        actions = ActionChains(self.driver)
        actions.move_to_element(element).perform()
```

```python
    def hover_with_offset(self, locator: str, x_offset: int, y_offset: int, by: str = By.XPATH) -> None:
        element = self.find(locator, by)
        ActionChains(self.driver).move_to_element(element).move_by_offset(x_offset, y_offset).perform()
```

- [ ] **Step 1: Write the pytester test with a stubbed `selenium` fixture**

Add to `tests/test_plugin.py` (no real Chrome â€” the stubbed `selenium` fixture returns a plain object so the fixture wiring is exercised without launching a browser):

```python
def test_jc_browser_wraps_selenium(pytester):
    pytester.makeconftest(
        """
        import pytest


        @pytest.fixture
        def selenium():
            return object()  # stub driver â€” jc_browser must wrap exactly this
        """
    )
    pytester.makepyfile(
        """
        from jc_selenium_helper.browser import Browser


        def test_wrap(jc_browser, selenium):
            assert isinstance(jc_browser, Browser)
            assert jc_browser.driver is selenium
        """
    )
    result = pytester.runpytest()
    result.assert_outcomes(passed=1)
```

## Notes for the implementer

- **Why the hover test passes before the fix (Task 1, Step 3):** M3.2 is a clarity refactor, not a behavior change â€” both the two-`perform()` and the single-chained forms end the pointer at element-center + offset. The test's job is to lock that observable contract so the refactor is provably safe, and to close the M0.2 coverage gap. Do not "make it fail first" by breaking the fixture; a green baseline here is correct and expected.
- **Fixture geometry is deterministic** only because `body{margin:0}` and both boxes use `position:absolute`. If you change the anchor/target coordinates, recompute so that (anchor-center + offset) still falls strictly inside the target box, with the boxes non-overlapping.
- **`jc_browser` test uses a stub, not Chrome.** Keep it that way â€” matching the rest of `tests/test_plugin.py`, which never launches a browser. Launching real Chrome in the pytester subprocess would need full pytest-selenium driver config and is out of scope.
