# M0.2 leftovers — `hover_with_offset` + `jc_browser` (bundling M3.2)

**Date:** 2026-07-05
**Status:** Design approved, pending implementation plan
**Audit ref:** `docs/superpowers/notes/2026-07-04-audit.md` — closes the two remaining
M0.2 test gaps and, because it is entangled with them, the M3.2 source fix.

---

## Problem

The v0.3.1 M0.2 test backfill covered most previously untested `Browser` methods, but
two items on the M0.2 list were never closed:

1. **`hover_with_offset`** (`browser.py:112`) — still untested. Its implementation uses
   two separate `perform()` calls, which is exactly the behavior the audit's **M3.2**
   flags as suspect. A meaningful behavior test must assert the final pointer position,
   which differs depending on whether M3.2's single-chained-action fix is applied — so
   the test and the fix cannot be sensibly separated.
2. **`jc_browser` fixture** (`plugin.py`) — still untested. `test_plugin.py` covers only
   the `chrome_options` fixtures.

## Decision

- **Bundle M3.2 with the `hover_with_offset` test** (owner-approved). Fix the source to a
  single chained action, then test the corrected behavior.
- **Test `jc_browser` via a stubbed `selenium` fixture** under `pytester` (owner-approved),
  matching the existing `test_plugin.py` pattern — no real Chrome.
- **Out of scope:** all other open audit items (M1.1, M2.1, M3.1, M3.3, M3.4) and any
  release / version bump. The CHANGELOG gains an unreleased entry only.

## Components

### 1. Source fix (M3.2) — `src/jc_selenium_helper/browser.py`

Replace the two-`perform()` body of `hover_with_offset` with a single chained action so the
offset is applied relative to the element center within one pointer sequence:

```python
def hover_with_offset(self, locator: str, x_offset: int, y_offset: int, by: str = By.XPATH) -> None:
    element = self.find(locator, by)
    ActionChains(self.driver).move_to_element(element).move_by_offset(x_offset, y_offset).perform()
```

No signature change. Behavior change: the final pointer position is now the element center
plus `(x_offset, y_offset)` in a single deterministic sequence.

### 2. New fixture — `tests/fixtures/offset.html`

A dedicated fixture (keeps coordinate-precise absolute positioning out of `dynamic.html`'s
normal-flow content). Requirements:

- `body { margin: 0 }` so absolute `left`/`top` equal viewport coordinates.
- `#anchor` — an absolutely-positioned box at a known position/size, so its center is
  deterministic (Selenium `move_to_element` targets the element center).
- `#offset-target` — a separate absolutely-positioned box placed so that
  *anchor-center + (x_offset, y_offset)* falls inside it. Sized generously (~60px) to
  absorb sub-pixel rounding. The two boxes must not overlap.
- A `mouseover` listener on `#offset-target` that writes the text `offset-hovered` into
  `#offset-result` on first hover (mirrors the `dynamic.html` `#hover-target` /
  `#hover-result` pattern).

Concrete layout (illustrative — exact values finalized in the plan): `#anchor` at
`left:100 top:100 width:40 height:40` → center `(120,120)`; offset `(100, 0)` → final
`(220,120)`; `#offset-target` at `left:190 top:90 width:60 height:60` covering
`x[190,250] y[90,150]`, which contains `(220,120)`.

### 3. Test — `tests/test_browser_actions.py`

`test_hover_with_offset_lands_on_offset_target`:

- `browser.open(fixture_url("offset.html"))`
- `browser.hover_with_offset("//div[@id='anchor']", 100, 0)`
- assert `browser.find("//div[@id='offset-result']").text == "offset-hovered"`

Asserts real observable behavior (a `mouseover` fired on the offset target), not merely
that the call did not raise — consistent with the M0.2 backfill style.

### 4. Test — `tests/test_plugin.py`

`test_jc_browser_wraps_selenium` (no real Chrome):

```python
def test_jc_browser_wraps_selenium(pytester):
    pytester.makeconftest(
        """
        import pytest

        @pytest.fixture
        def selenium():
            return object()  # stub driver
        """
    )
    pytester.makepyfile(
        """
        from jc_selenium_helper.browser import Browser

        def test_wrap(jc_browser, selenium):
            assert isinstance(jc_browser, Browser)
            assert jc_browser.driver is selenium
        """
    )
    result = pytester.runpytest()
    result.assert_outcomes(passed=1)
```

Verifies the exact wiring contract: `jc_browser` returns a `Browser` wrapping the precise
`selenium` object it was given.

### 5. CHANGELOG — `CHANGELOG.md`

chained action (previously two separate `perform()` calls), plus a note that the internal
test suite now covers `hover_with_offset` and the `jc_browser` fixture. No version bump.
