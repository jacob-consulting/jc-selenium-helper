# M2.1 — Coverage floor

**Date:** 2026-07-05
**Status:** Design approved, pending implementation plan
**Audit ref:** `docs/superpowers/notes/2026-07-04-audit.md` — closes M2.1, the last open
audit item. Resolves the audit's Open Question #4 (floor target): **80% package-wide, at the
current honest baseline.**

---

## Problem

M0.1 turned on coverage reporting in CI (`pytest --cov=jc_selenium_helper`), but the number
**under-counts**. The `pytest11` entry point (`jc_selenium_helper.plugin`) is loaded at pytest
startup, which imports the package — and `src/jc_selenium_helper/__init__.py` does
`from jc_selenium_helper.browser import Browser` at module top. That import runs **before**
pytest-cov starts coverage, so coverage emits
`CoverageWarning: Module jc_selenium_helper was previously imported, but not measured
(module-not-measured)` and the package's import-time lines read as missed. Observed vs. true:

| Module | Reported (pytest --cov) | True (coverage run) |
|---|---|---|
| `__init__.py` | 0% | 100% |
| `browser.py` | 65% | 96% |
| `plugin.py` | 45% | 100% |
| `legacy.py` | 44% | 44% |
| `axe/colors/config.py` | 100% | 100% |
| **TOTAL** | 60% | **79.78%** |

A coverage floor set on the reported number would be meaningless. The measurement must be
fixed first.

## Decision

- **Fix measurement by running coverage as the outer process:** `coverage run -m pytest`
  starts measurement before pytest loads any plugin, so the entry-point import is measured.
  This needs **no source change** (verified: the `module-not-measured` warning disappears and
  totals reach their true values).
- **Floor: 80% package-wide** (owner-approved). The true total is 79.78%, which coverage
  displays and compares as `80` at default precision (0), so `fail_under = 80` passes today and
  trips on a ~2-statement regression. The 79.78% baseline is deterministic across the
  3.12/3.13/3.14 matrix (the missed lines — `browser.py` 72-75/106 and `legacy.py`'s untested
  compat-shim delegators — are version-independent).
- **No new tests.** `legacy.py` stays at 44%; the floor is set at the current honest baseline,
  not raised by backfilling. That keeps M2.1 focused; a legacy backfill is a separate concern.

## Components

### 1. Coverage config — `pyproject.toml`

Add:
```toml
[tool.coverage.run]
source = ["jc_selenium_helper"]

[tool.coverage.report]
show_missing = true
fail_under = 80
```
`source = ["jc_selenium_helper"]` maps the editable install back to `src/jc_selenium_helper/`
(confirmed by the existing report's `src/...` paths). coverage 7.x reads this from
`pyproject.toml` natively (stdlib `tomllib` on Python ≥3.11). `fail_under` makes
`coverage report` exit non-zero below the floor.

### 2. Dependency swap — `pyproject.toml` `test` extra

The `test` extra (lines 39-44) currently lists `"pytest-cov"`. Since CI no longer uses
pytest-cov's `--cov` plugin, replace it with `"coverage[toml]>=7"`:
```toml
test = [
    "jc-selenium-helper[pytest,config,axe]",
    "nox",
    "coverage[toml]>=7",
    "pytest-xdist",
]
```
Honest dependency for what CI now does; drops the unused plugin.

### 3. CI — `.github/workflows/ci.yml`

Replace the single step (line 32):
```yaml
      - run: pytest -q --cov=jc_selenium_helper --cov-report=term-missing
```
with two steps:
```yaml
      - run: coverage run -m pytest -q
      - run: coverage report
```
`coverage report` reads `fail_under` from config; below the floor it exits 2 and fails the job.
Runs on each matrix Python (3.12/3.13/3.14). The `addopts = "-p no:cacheprovider"` in
`[tool.pytest.ini_options]` still applies (pytest reads it under `coverage run -m pytest`).

### 4. `.gitignore`

`coverage run` writes a `.coverage` data file; `.gitignore` does not currently list it. Add it
under the "Tooling caches" group:
```
.coverage
```

### 5. CHANGELOG — `CHANGELOG.md` (existing `## Unreleased` section)

Add an **Internal** bullet (no version bump):
```markdown
- Coverage is now measured via `coverage run -m pytest` (fixing an import-timing
  under-count from the pytest entry point) with an 80% floor enforced in CI.
```

## Testing / verification

- `coverage run -m pytest -q` → 56 passed; the run log contains **no** `module-not-measured`
  warning.
- `coverage report` → TOTAL 80% (79.78% at full precision), exit 0 (floor satisfied).
- One-off sanity (NOT committed): `coverage report --fail-under=95` exits 2 — proves the gate
  bites. `coverage report --fail-under=80` exits 0 — proves the chosen floor passes.

## Acceptance criteria

- `pyproject.toml` has `[tool.coverage.run]` (`source`) and `[tool.coverage.report]`
  (`show_missing`, `fail_under = 80`); `test` extra lists `coverage[toml]>=7`, not `pytest-cov`.
- `ci.yml` runs `coverage run -m pytest -q` then `coverage report`; the coverage floor fails
  the job when breached.
- `.gitignore` ignores `.coverage`.
- Local `coverage run -m pytest -q && coverage report` passes at ~80% with no
  `module-not-measured` warning; full suite still 56 passed.
- CHANGELOG has the unreleased Internal entry. No version bump.

## Open questions

None — fix approach (`coverage run -m pytest`), floor (80% package-wide baseline), and the
`pytest-cov` → `coverage[toml]` swap are all resolved.
